# Idea: Improve README to Better Sell Unique Value

## One-Line Overview
Rewrite the repo's README to clearly articulate what makes this roadmap convention uniquely better than existing alternatives, using real analysis to back the claims.

## What's the idea

The current README explains *what* the convention is and *how* to use it, but doesn't make a strong case for *why* someone would pick it over other roadmap approaches. The idea is to sharpen the README with a clear differentiated value proposition — grounded in an analysis of alternatives.

## Expected advantages / benefits

- Faster adoption: visitors immediately understand why this is worth using
- Reduces the "why not just use GitHub Issues / Notion / Linear?" objection
- Positions the convention as AI-native, not just another markdown system

## Downsides / risks

- Risk of over-claiming — needs to stay honest and specific, not marketing fluff
- README could get too long if we try to cover everything

## What's been tried already

Analysis of this repo vs. alternatives has been done — see `docs/` for source material.

## Open questions

1. What's the single sharpest differentiator — AI-nativeness, simplicity, or co-location with code?
2. Should the README lead with the problem or the solution?
